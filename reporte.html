Gkals Habits PWA

PWA móvil para iPhone, enfocada en programar hábitos, marcar cumplimiento diario y repetir recordatorios cuando un hábito queda vencido.

📱 ✅ 🔔
⚛️ React + Vite + TS 🗄️ Supabase 🔔 Web Push 🍎 iPhone-first

🧱 Stack tecnológico

⚛️
React + Vite + TypeScript
Frontend moderno con tipado estático y build ultrarrápido.
🗄️
Supabase
Auth, Postgres y Edge Functions como backend completo.
🔔
Service Worker propio
Notificaciones con Web Push gestionadas a medida.
📱
UI mobile-first
Diseñada para iPhone con soporte de safe areas.

🚀 Arranque

npm install
npm run icons
cp .env.example .env
npm run dev

Configura en .env:

VITE_SUPABASE_URLURL del proyecto Supabase
VITE_SUPABASE_ANON_KEYClave anónima pública de Supabase
VITE_VAPID_PUBLIC_KEYClave pública VAPID para Web Push

🗄️ Configuración de Supabase

  1. Crea un proyecto en Supabase.
  2. Ejecuta supabase/schema.sql en el SQL Editor.
  3. En Authentication, desactiva la confirmación de email para que las cuentas se creen sin verificación.
  4. Copia la URL y la anon key al .env.

Para los recordatorios push, configura los secretos y despliega la función:

supabase secrets set SUPABASE_SERVICE_ROLE_KEY=...
supabase secrets set VAPID_PUBLIC_KEY=...
supabase secrets set VAPID_PRIVATE_KEY=...
supabase secrets set VAPID_SUBJECT=mailto:ops@example.com
supabase secrets set CRON_SECRET=...
supabase functions deploy send-habit-reminders
⏰
Programa la función cada 10 minutos desde Supabase Scheduler o un cron externo, enviando el header: 
Authorization: Bearer $CRON_SECRET

🍎 iPhone y Web Push

⚠️
Para Web Push en iOS, la app debe estar instalada desde Safari en la pantalla de inicio y servida por HTTPS.

La persistencia se implementa con tres mecanismos:

🔁
Repetición en backend
Los recordatorios se reenvían desde el servidor hasta que el hábito se cumple.
📌
requireInteraction
La notificación permanece visible cuando el navegador lo soporta.
👁️
Deuda visible
Los hábitos vencidos se mantienen a la vista dentro de la app.

💬 Tono de los mensajes

El lenguaje agresivo queda centralizado en src/lib/motivation.ts y en la función send-habit-reminders. Cambiar la personalidad no requiere tocar las pantallas.
